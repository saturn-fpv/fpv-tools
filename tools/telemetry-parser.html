Telemetry Parser

📖 Overview

Convert EdgeTX radio .csv telemetry logs (CRSF or ELRS) into Google Earth 3D KML/KMZ and GPX flight tracking files. All processing runs 100% locally in your browser — log files are parsed client-side and never uploaded to any external server.

🛠️ How to Use

  1. Browse or drag-and-drop your EdgeTX .csv log file.
  2. Select a flight segment from the auto-detected list.
  3. Configure altitude offset, track width, and line color.
  4. Inspect the interactive map preview and speed/altitude profiles, then click Download KML/KMZ.

⚙️ Parser Options & Export Formats

  • Flight Segment Selector: Auto-detects and isolates separate flights from a single log file based on arm/disarm state and power cycles.
  • Altitude Offset (m): Adjusts baseline ground elevation zero-level if barometric sensor or GPS altitude reported an offset.
  • Track Line Color & Width: Customizes the visual 3D path ribbon rendered in Google Earth.
  • KML/KMZ vs GPX Export: KML/KMZ exports 3D flight paths, altitude extrusion ribbons, and waypoints for Google Earth. GPX exports standard GPS track logs compatible with Strava, Garmin, and GIS tools.

⚙️ EdgeTX Logging Setup Guide

  1. On EdgeTX Telemetry screen, click Discover new sensors (verify GPS coordinates).
  2. In Special Functions (e.g. SF1), assign your ARM switch to action SD Logs.
  3. Set interval to 1.0s (or 0.5s) and check the enable box.

💡 GPS Smoothness & Telemetry Ratios

If GPS tracking appears choppy or updates infrequently (every 5–8s), your ExpressLRS Telemetry Ratio is set too low (e.g. 1:64). Open the ExpressLRS LUA script on your radio and set Telem Ratio to 1:8 or 1:4 for smooth high-frequency logging.
